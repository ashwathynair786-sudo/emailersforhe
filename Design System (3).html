Brand fundamentals.

Three things define the HackerEarth visual system: a warm cream surface, a deep navy / orange accent pair, and an editorial display typeface. Everything else flows from those three decisions.

The wordmark

The HackerEarth wordmark is always lowercase, set in the brand's custom typeface, and rendered in brand black #020109. Use at 22px tall in nav contexts; scale proportionally elsewhere. Minimum size: 18px height. Don't lock it up with other elements, don't add a tagline, don't recolor it.

Color.

A 7-color brand palette plus 3 functional state colors. Two saturated accents (navy, orange) anchored by warm and cool neutrals, with a third accent (lime) reserved for highlights. Use restrained.

Typography.

Three typefaces, each with one job. Editorial display for headlines. Neutral grotesque for body. Monospace for technical labels and eyebrows.

Display — Bricolage Grotesque

The voice of the brand. Used for every headline, statistic, and section title. Always at weight 500 (medium). Variable optical-size axis dialed up for display sizes. Emphasis is rendered in the pop color at the same weight — never italic, never bold.

Pop-color emphasis pattern

Every headline that takes emphasis uses a one-phrase pop in orange — same weight as the rest of the headline, just recolored. Never italic. Never bold. The pop color does the work alone. One emphasis per headline, max. Wrap the phrase in <em> with font-style: normal.

Components.

The recurring patterns. Build with these — don't invent new ones for one-off uses.

Buttons

One primary, one secondary. Primary uses brand orange and only ever drives the page's main action ("Book a demo", "Talk to research"). Secondary is text + underline. There is no tertiary or ghost button.

Eyebrows

Eyebrows pair a small compass-rose mark, a bracketed section number, and an all-caps tracked label. No pill background — the compass and the bracket carry the structure. On cream, the compass N-pointer is brand orange. On dark, lime takes over. Mono font, 11px, 1.5px letter-spacing.

Layout & spacing.

Consistent rhythm makes the system feel intentional. Don't invent new values mid-page.

Section padding

100px top + bottom is the default for content sections. 120px for bottom-CTA sections (extra breathing room around the ask). 80px for tighter sections like social proof. Always vertical-only — horizontal padding lives on the inner .container.

Surface alternation

Cream and dark surfaces alternate to create visual rhythm. A good rule of thumb: 2–3 cream sections, then one dark section. Don't stack two dark sections back-to-back. Cards stay white regardless of which surface they sit on.

Voice & emphasis.

The visual system has a voice. These are the patterns that make a page sound like HackerEarth even before someone reads the copy.

Pop-color emphasis headlines

Every major headline has one phrase recolored in pop orange. This is the most identifiable typographic signature. The colored phrase is the active idea; the surrounding text frames it. One emphasis per headline — never two, never italic, never bold.

  • Phone screens don't scale.
  • A real interview, for every candidate.
  • Skills evaluation, built like infrastructure.
  • Your next hire is in the candidates you're missing.

Eyebrow language

Eyebrows are short. 2–5 words. They're tracking labels, not sentences. Style: short, definite, plain.

Good: Inside OnScreen · The platform · By the numbers · For developers
Bad: Discover how HackerEarth helps you... · Our incredible features for modern teams

Stat construction

Always pair a stat with a tight label. Number bold, unit dim, label one line. No more than 3 stats per row, no more than 4 per section. Stats with units larger than 3 characters (yrs, mo, %, ×, M, B, K) should be at ~55% opacity-and-size compared to the number itself.

Contours.

Cartographic contour rings are the brand's signature background motif. Use them optionally as section backdrops to reinforce the topographic identity — never on every section, never behind body copy that needs to be read at speed.

When to use

Reserve contours for: hero backdrops, footer backgrounds, deep-section dividers, and product-mock surroundings (never over the UI). Always pair with a radial mask that fades them toward the text. Never use as foreground content.

On cream surfaces

Navy strokes at varying opacity. Lime accent on the inner peak when a focal point is desired; otherwise omit the accent ring.

On cream · navy strokes + orange accent peak

On dark surfaces

White strokes at varying opacity. Lime accent on the focal peak. Same shapes, inverted colors.

On dark · white strokes + lime accent peak

Token specification

Six tokens drive every contour treatment. Use these values, don't invent intermediate opacities.

On cream — strokes

  • rgba(19,37,93, 0.22) · default ring
  • rgba(19,37,93, 0.36) · .major (every 3rd ring)
  • rgba(255,87,34, 0.45) · .cta (focal peak)

On dark — strokes

  • rgba(255,255,255, 0.18) · default ring
  • rgba(255,255,255, 0.28) · .major (every 3rd ring)
  • color-mix(--lime 55%, transparent) · .lime (focal peak)

Mask gradients

Contours must fade so they never compete with content. Use a radial mask that fades the rings toward the headline area.

/* Rings clustered on the right; fade toward headline on left */
.topo-bg-dark.fade-left::after {
  content: '';
  position: absolute;
  inset: 0;
  background: radial-gradient(
    ellipse 55% 75% at 18% 50%,
    var(--bg-dark) 0%,
    var(--bg-dark) 28%,
    transparent 75%
  );
}

/* Mirror to flip horizontally when needed */
.topo-bg-dark.mirror svg {
  transform: scaleX(-1);
  transform-origin: center;
}

Stroke widths

Default stroke width is 0.9. .major rings (every 3rd) bump to 1.1. The focal-peak ring (.cta or .lime) uses 0.9 but at higher opacity — the color does the lifting, not the weight.

Section opacity

Cap the parent backdrop opacity at 0.5 on cream and 0.32–0.45 on dark. Higher than this and the rings start fighting the headline. Lower than 0.3 and they vanish. The system has a narrow legal range; stay inside it.

Companion: soft mist (Platform pattern)

For sections where contour rings would be too literal, use the soft-mist variant — heavily blurred elliptical blobs in dark gray, no strokes. Same earth motif, different render. Best for sections that already carry information (architecture diagrams, deep-dive panels).

Do / don't.

The rules that keep the system from drifting.

Do

  • Use orange (--cta) only for primary CTAs and one emphasized phrase per headline.
  • Reserve lime for 1–2 highlight moments per page — badges, "new," featured borders.
  • Use the cream (#FBF9F3) — never a generic off-white. Warmth is part of the brand.
  • Use white stat numbers on dark; mute the unit with smaller size and gray.
  • Use a single accent at varying intensity for layered diagrams.
  • Set eyebrows in mono, uppercase, tracked.
  • Alternate cream and dark sections for rhythm.
  • Use real customer logos at 70% opacity, grayscale.
  • Recolor one phrase per headline in brand orange — same weight, never italic.
  • Mark placeholder content visibly (dashed borders, dimmed opacity).

Don't

  • Don't use orange as a category color or decorative fill — it's an action color.
  • Don't use lime more than twice per page. Don't use it as paragraph color.
  • Don't introduce hex codes that aren't in this spec. Use color-mix instead.
  • Don't use periwinkle (#9FB1FE) or lavender (#C7B4F8) — these are old palette and have been removed.
  • Don't bold or italicize for emphasis — orange recolor only.
  • Don't use rainbow categories on diagrams. One color, varying intensity.
  • Don't use emoji.
  • Don't use Inter, Roboto, system-ui as defaults — they're fallbacks only.
  • Don't use animated mesh gradients on the hero. (Toggleable opt-in only.)
  • Don't use a scrolling marquee for customer logos.
  • Don't stack two dark sections back-to-back.
  • Don't invent new spacing values mid-page — stick to the scale.
